# Archived strategy documents

**Purpose:** Store **long-horizon or aspirational** material so it does not compete with **near-term execution truth** in the main docs tree.

## What belongs here

- Multi-year **vision** scenarios, investor-style projections, or brainstorming that **has not** been validated against current operations.
- Documents that may still be **useful** for leadership off-sites or fundraising drafts but must **not** be mistaken for **current engineering scope**, **committed metrics**, or **PLATFORM_SOURCE_OF_TRUTH** decisions.

## What is *not* archived

| Document | Location | Role |
|----------|----------|------|
| **Strategic foundation** (onboarding, theory of change, ~12-month honesty) | [../STRATEGIC_FOUNDATION.md](../STRATEGIC_FOUNDATION.md) | **Default** read for “why we exist” and holistic problem framing. |
| **Platform decisions** (auth, reports, priorities) | [../PLATFORM_SOURCE_OF_TRUTH.md](../PLATFORM_SOURCE_OF_TRUTH.md) | **Binding** technical/product spec. |

## Contents

| File | Notes |

**Rule:** If a number in an archived doc could embarrass the organisation if quoted as “committed,” do **not** cite it in code comments, public README, or partner decks without CEO approval.
